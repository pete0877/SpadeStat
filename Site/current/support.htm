Support & FAQ

If you are having problems installing or using the SpadeStat software, please send us email at: support@spadestat.com. Please include details on what your problems are.

We also have a Blog available at: http://spadestat.blogspot.com


Here some commonly asked questions:

Question: What are the system requirements for SpadeStat?

Answer:

    Supported Operating Systems: Windows XP, Windows 2000, Windows NT, Windows 2003.
    Recommended hardware configuration: 800+ Mhz CPU, 256+ Mb of RAM memory, 10+ Gb of free disk space.


Question: What types of games do you support and are there plans for other games?

Answer:

    Currently, SpadeStat can handle hand histories from Hold'em, Omaha, Omaha Hi/Lo, 7-Card Stud and 7-Card Stud Hi/Lo. SpadeStat supports all limit, pot limit and no limit versions of these games. As we develop support for other poker sites, other games will be added to SpadeStat's abilities.


Question: What sites do you support and when will other sites become available?

Answer:

    Currently, SpadeStat can handle hand histories and tournament results from Pokerstars. Development is underway to expand SpadeStat's capabilities for other sites. Currently, Party Poker and its skins are the next site to be expanded into. After that, there are a number of sites on our lists, but it will depend upon the desires of our customer.


Question: Will you ever support real-time importing of data off of poker site client software?

Answer:

    As this is known to be a desired feature for many, yes, we will strive to make this possible. It has not been determined when this feature will be implemented, but we will keep everyone appraised of our progress as developments are made.


Question: How do I use the Database Query section? What is SQL?

Answer:

    The database query tab is for advanced users only and really only for people who understand the syntax of the SQL query language. There are plenty of resources to teach someone how to use SQL query language, and we hope that some of our users will research them. SQL stands for Standard Query Language and is a database standard for most available databases. SpadeStat uses PostGreSql - which is merely an open-source variant of SQL.


Question: I get an error while I import a hand or tournament file, what should I do?

Answer:

    The first thing to do is to write down any sort of error information that you receive and immediately contact SpadeStat support. We will work you through the problem as best as we can. When contacting us, make sure you attach the file that produced the error and any and all information relating to it.


Question: I get an error while running a report, what should I do?

Answer:

    The first thing to do is to write down any error information that you receive and immediately contact SpadeStat support. We will work you through the problem as best as we can. When contacting us, make sure you include the steps you went through to receive the error and the name of the report(s) that were producing errors.
